## Person 1 — Fix Two Java Bugs in the Backend
**File:** `src/service/mallKioskService.java`

### Bug A — Typo in method name (line 245)

**Find this line:**
```java
public Receipt generaReceipt(List<StoreItem> outfit, int sessionID)
```

**Change it to:**
```java
public Receipt generateReceipt(List<StoreItem> outfit, int sessionID)
```

**Why:** The method is missing the letter `t` — it says `generaReceipt` instead of `generateReceipt`. Java method names must match exactly when called from other places, so this typo would cause a "method not found" compile error if anyone tries to call `generateReceipt(...)`.

```java

Switching to `databaseConnection.getConnection()` makes it consistent with the rest of the file and actually work.

Persons 2 and 3 must coordinate: both should use the exact same parameter name `gender`.

### Step 2 — Update `fetchOutfit` to flip the loading flag

**Find the entire `fetchOutfit` function (lines 26–47):**
```jsx
async function fetchOutfit() {
  try {
    const params = new URLSearchParams();

    if (occasion) {
      params.append("occasion", occasion);
    }

    const url = `http://localhost:8080/api/outfit?${params.toString()}`;

    console.log("Fetching outfit from:", url);

    const res = await fetch(url);

    console.log("Received outfit:", data);

  } catch (error) {
    console.error("Failed to fetch outfit:", error);
  }
}
```

**Replace with:**
```jsx
async function fetchOutfit() {
  setLoading(true);
  try {
    const params = new URLSearchParams();

    if (occasion) {
      params.append("occasion", occasion);
    }

    const url = `http://localhost:8080/api/outfit?${params.toString()}`;
    const res = await fetch(url);
    const data = await res.json();

    setOutfit(data);
  } catch (error) {
    console.error("Failed to fetch outfit:", error);
  } finally {
    setLoading(false);
  }
}
```
